# Votelist 2026 — Current Working Structure

Static GitHub Pages campaign/resident management app for Villimale.

Live site:

```text
https://naappe.github.io/votelist2026/
```

Repository:

```text
naappe/votelist2026
```

Backend:

```text
Supabase project: espezmdpkoixnfchomqb
Main table: public.campaign
```

This app is a browser-only website. There is no Node, Python, or backend server inside GitHub. GitHub Pages serves HTML/CSS/JS, and the browser talks directly to Supabase.

---

## 1. Current System Rule

The current app has been cleaned to use one main working page:

```text
residents.html
```

Most old pages now redirect into `residents.html` with a `section` parameter.

Example:

```text
assign.html?party=MDP
```

redirects to:

```text
residents.html?party=MDP&section=assign
```

This was done because old pages had duplicate scripts, old login checks, old design, and conflicting logic.

## 6. Current Functions in `residents-loader.js`

### 6.1 Load Data

Loads rows from:

```text
public.campaign
```

If URL has `party=PNC`, only PNC rows are loaded.

If URL has `party=MDP`, only MDP rows are loaded.

If URL has `party=ALL`, all rows are loaded.

The loader uses pagination/ranges so it can load more than 1000 rows.

---

### 6.2 Stats

Stat cards show counts for the currently loaded party scope.

| Stat | Meaning |
|---|---|
| Total Residents | All loaded rows. |
| Will Vote | `vote_status = will-vote`. |
| Not Vote | `vote_status = no-vote`. |
| Reached Pending | Reached residents without final vote result. |
| Need Call / Visit | Not reached, need call, or not visited. |

---

### 6.3 Clickable Stats / Filtering

Clicking a stat filters the list.

| Click | Filter result |
|---|---|
| Total Residents | Shows all loaded rows. |
| Will Vote | Shows only `vote_status = will-vote`. |
| Not Vote | Shows only `vote_status = no-vote`. |
| Reached Pending | Shows reached residents with no final vote. |
| Need Call / Visit | Shows rows where call/visit work is still needed. |

Search and house filters apply on top of the selected stat filter.

---

### 6.4 Search

Search checks these fields:

```text
name
national_id
house
phone
party
election_box
remarks
```

---

### 6.5 House Filter

The house dropdown is built from the loaded rows.

Selecting a house filters the current view to only that house.

---

### 6.6 Clear Search

The clear button:

```text
- clears search text
- clears house dropdown
- resets stat filter to Total / All
```

---

### 6.7 Edit / Save Update

Clicking a resident card opens the update panel.

Editable fields:

```text
vote_status
phone_status
reach_status
d2d_status
transport_status
support_level
remarks
```

Save writes back to:

```text
public.campaign
```

Auto logic:

```text
will-vote => reached
no-vote => reached
guaranteed => reached
called => reached
```

## 7. Supabase Security / RLS Notes

Current situation:

```text
SELECT: public read is enabled
UPDATE: working fields are allowed for anon/authenticated updates
```

This was necessary because the current clean system has no login screen.

Only working fields should be updated from the public page. Do not expose delete, table reset, or unrestricted SQL from the frontend.

Important: Removing login makes the page easier to use, but anyone with the link can view and update allowed working fields. If stricter security is needed later, restore login and authenticated RLS.

## 8. Why the Errors Happened

### Error: Page loads but data does not load

Common causes:

```text
- js/config.js not loaded
- Supabase CDN not loaded
- wrong table name
- RLS SELECT policy blocked anon/public read
- browser cache using old JS
```

Check Console:

```js
console.log(window.APP_CONFIG)
console.log(window.supabase)
```

---

### Error: Stats show but clicking does nothing

Cause:

```text
Stats were only visual numbers. They did not have click handlers.
```

Fix:

```text
residents-loader.js now binds stat cards to filters.
```

---

### Error: List loads but Save Update fails

Common causes:

```text
- UPDATE RLS policy blocks anon user
- update grant missing
- invalid value not allowed by table CHECK constraint
- browser is using cached old script
```

Example invalid values:

```text
d2d_status = pending   // invalid in current campaign table
transport_status = none // invalid in current campaign table
```

Use only allowed values from the table checks.

---

### Error: Old page opens / wrong design appears

Cause:

```text
Old route file was still loading old scripts.
```

Fix:

```text
Old route files now redirect to residents.html with section parameters.
```

## 12. Future Developer Rules

1. Keep `residents.html` as the main app page.
2. Keep old route files as redirects unless intentionally rebuilding them.
3. Do not duplicate old login/dashboard scripts into the clean page.
4. Do not change Supabase voter/resident data unless the task specifically says to update data.
5. Do not use invalid status values that violate table CHECK constraints.
6. Update this README whenever routes, table names, status values, or major scripts change.
